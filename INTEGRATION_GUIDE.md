## How to Integrate AI-Generated Components

### 1. UI Components (Button, Card, Input, etc.)

If your AI tool generates a UI component like this:

```tsx
// AI-generated button.tsx
import { cn } from "@/lib/utils"

export function Button({ className, ...props }) {
  return <button className={cn("...", className)} {...props} />
}
```

**Integration Steps:**
1. Create a new file in `src/components/ui/` (e.g., `dialog.tsx`, `dropdown.tsx`)
2. Update the import path from `@/lib/utils` to `../../lib/utils`
3. Save the file
4. Export it from your screen/page component

**Example:**
```tsx
// src/components/ui/dialog.tsx
import { cn } from "../../lib/utils";  // â† Changed from @/lib/utils

export function Dialog({ className, ...props }) {
  return <div className={cn("...", className)} {...props} />
}
```

### 2. Page/Screen Components

If your AI tool generates a full page:

```tsx
// AI-generated landing-page.tsx
import { Button } from "@/components/ui/button"

export default function LandingPage() {
  return <div>...</div>
}
```

**Integration Steps:**
1. Create a folder in `src/screens/` (e.g., `src/screens/LandingPage/`)
2. Create `LandingPage.tsx` and `index.ts` files
3. Update import paths:
   - `@/components/ui/button` â†’ `../../components/ui/button`
   - `@/lib/utils` â†’ `../../lib/utils`
4. Export from `index.ts`
5. Import in `src/App.tsx`

**Example:**
```tsx
// src/screens/LandingPage/LandingPage.tsx
import { Button } from "../../components/ui/button";  // â† Updated path
import { Card } from "../../components/ui/card";      // â† Updated path

export const LandingPage = () => {
  return <div>...</div>
}
```

```tsx
// src/screens/LandingPage/index.ts
export { LandingPage } from "./LandingPage";
```

```tsx
// src/App.tsx
import { LandingPage } from "./screens/LandingPage";
import "./index.css";

function App() {
  return <LandingPage />;
}

export default App;
```

### 3. Common Path Aliases to Replace

| AI-Generated Path | Replace With |
|------------------|--------------|
| `@/lib/utils` | `../../lib/utils` |
| `@/components/ui/button` | `../../components/ui/button` |
| `@/components/ui/*` | `../../components/ui/*` |

## Tips

- Keep UI components generic and reusable in `src/components/ui/`
- Keep page-specific components in their respective screen folders
- Use the `cn()` utility function for conditional className merging
- Follow the existing naming conventions for consistency
- Test each component after integration
